# 嗨嗨聊天室（hihi）

匿名、閱後即焚的**公開主題聊天室**。對標 knock.tw（敲敲看），但**乾淨、合法、安全為第一原則**。

## 定位

- **乾淨的公開門面 + 新流量入口**：一個獨立、合法、低風險的匿名聊天產品，當作生態系的乾淨門面與獲客入口（分散「只靠腳本一條腿」的風險）。
- **安全設計**（守住「乾淨合法安全」，避免匿名聊天變成 moderation 黑洞）：
  - **純文字，不接受任何連結**（貼連結一律擋 → 掐掉廣告/違法散布主途徑）
  - **違法/高風險關鍵字直接攔**（未成年/性影像/交易/毒品/暴力徵求）
  - **閱後即焚**：訊息不落地，只在記憶體保留每房最近數則給新加入者
  - **每連線限流**（擋洗版/機器人）
  - **公開主題房**而非 1 對 1（公開＝可見、自我約束，比私密 1 對 1 難出事）
  - **進聊要驗證**：預留 areyoubot（PoW CAPTCHA）接點（`chat.js` join 的 `verify` 欄位）

## 架構

```
server.js       http 服務 public/ + WebSocket(/ws)
chat.js         聊天核心：主題房、匿名、廣播、閱後即焚、套 filter+ratelimit
lib/
  filter.js     訊息過濾（擋連結/違法/廣告 + HTML escape）
  ratelimit.js  每連線滑動視窗限流
  env.js        無依賴 .env 載入器
public/index.html  匿名進入 + 房間 + 聊天 UI（低調乾淨）
```

## 跑起來

```bash
npm install
npm start        # http://localhost:4030
```

## 路線

- **P1**（現在）：公開主題房匿名聊天、過濾、限流、閱後即焚 — 可跑的骨架
- **P2**：接 areyoubot「要聊要驗證」、檢舉+自動下架、封鎖(IP/暱稱)、更完整的違法字庫
- **P3**：把 knock.tw 的 UX 細節補齊（可能加隨機 1 對 1 配對，但需更強 moderation）、lurlhub 側門導流（低調）
- **部署**：獨立網域 + 境外/獨立主機，跟成人站與實名資產切開（乾淨門面才乾淨）
